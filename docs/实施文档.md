# 智能气垫椅 — 触摸屏温控 实施文档（一步步操作）

> 配套：[需求文档.md](需求文档.md)
> 目标读者：第一次用这块带屏 ESP32，按顺序照做即可。
> 板子：**Sunton ESP32-3248S035R**（黄色屏 / 3.5" ST7796 / XPT2046 电阻触摸 / ESP32-WROOM-32E）

整体分 5 步，**强烈建议按顺序来**，尤其先把屏幕点亮再接线：

1. 装环境（Arduino IDE + ESP32 + 库）
2. 点亮屏幕自检（不接任何线）
3. 接线（DS18B20 + 4 路 MOS）
4. 烧录正式固件 + 触摸校准
5. 调试与验收

---

## 第 1 步：装开发环境

### 1.1 安装 Arduino IDE
到 https://www.arduino.cc/en/software 下载安装 Arduino IDE 2.x。

### 1.2 添加 ESP32 开发板支持
1. 打开 IDE → `File / 文件` → `Preferences / 首选项`
2. 在 **Additional boards manager URLs / 附加开发板管理器网址** 填：
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
3. `Tools / 工具` → `Board / 开发板` → `Boards Manager / 开发板管理器` → 搜 **esp32** → 安装 **esp32 by Espressif**

### 1.3 选择开发板
`Tools / 工具` → `Board / 开发板` → `ESP32 Arduino` → 选 **ESP32 Dev Module**
> ⚠️ 是 **ESP32 Dev Module**，不是 "ESP32S3"！你这块是老款 ESP32。

### 1.4 安装三个库
`Tools / 工具` → `Manage Libraries / 库管理器`，分别搜索并安装：
- **LovyanGFX**（作者 lovyan03）— 屏幕 + 触摸
- **OneWire**（作者 Paul Stoffregen）— DS18B20 通信
- **DallasTemperature**（作者 Miles Burton）— DS18B20 读温度

> `Preferences` 库是 ESP32 核心自带的，不用单独装。

---

## 第 2 步：点亮屏幕自检（先别接任何线！）

这一步只插 USB-C，确认屏幕配置正确。**配置不对就在这里改，别等接完线再排查。**

1. 用数据线把板子 USB-C 接到电脑。
2. `Tools / 工具` → `Port / 端口` 选出现的串口（Mac 上类似 `/dev/cu.usbserial-xxxx` 或 `/dev/cu.wchusbserialxxxx`；如果没有端口，多半缺 **CH340 驱动**，搜 "CH340 驱动 Mac/Windows" 装上）。
3. 打开 [code/00_screen_test.ino](../code/00_screen_test.ino)，点左上角 **→（Upload）** 烧录。
4. 烧录完，屏幕应显示：
   - 绿字 `SCREEN OK`
   - 下方三个色块：**红 / 绿 / 蓝**（从左到右）
   - 用手指点屏幕，串口监视器（115200 波特率）打印 `touch x=.. y=..`，点的位置出现黄点

### 自检结果判断

| 现象 | 处理 |
|------|------|
| 全部正常 | ✅ 进入第 3 步 |
| 屏幕全黑、背光也不亮 | 检查 USB 线是否数据线、端口是否选对；重新烧录 |
| 有背光但花屏/白屏 | SPI 配置问题，一般本配置 OK；可把 `freq_write` 从 40000000 降到 27000000 再试 |
| 颜色反相（背景发白） | 把 00 和 03 里 `cfg.invert` 改成 `false` |
| 红蓝互换（色块变成 蓝/绿/红） | 把 `cfg.rgb_order` 改成 `true` |
| 触摸点位置和手指差很远 | 正常，电阻屏要校准，第 4 步会做 |

> 颜色相关的两个开关（`invert` / `rgb_order`）在 00 和 03 两个文件里都有，改的话两边保持一致。

## 第 3 步：接线

> ⚠️ **接线前先拔掉 USB**，断电操作。

所有外设都接到板子的 **I2C 4pin 座子**，丝印是 `3.3V / IO32(SDA) / IO25(SCL) / GND`。
（这块板的 JST 1.25 座子需要对应的 4pin 杜邦/转接线，没有的话可以飞线焊。）

### 3.1 DS18B20 温度传感器

```
DS18B20         ESP32 板 (I2C座子)
  VCC(红)  ─────  3.3V
  GND(黑)  ─────  GND
  DATA(黄) ─────  IO25
                   │
         4.7kΩ 上拉│ (DATA 和 3.3V 之间接一个 4.7k 电阻)
  3.3V ──[4.7kΩ]──┘
```
> DS18B20 必须在 DATA 与 3.3V 之间接 4.7kΩ 上拉电阻，否则读不到温度（读出 -127 或 85）。

### 3.2 四路 MOS 加热模块

4 个 MOS 模块的**信号输入端（常标 PWM / SIG / IN）全部并联**到 **IO32**；4 个模块的逻辑地都接板子 GND（共地）：

```
        ┌──────────── IO32 (一根信号线分出 4 路)
        │
   ┌────┼────┬────────┬────────┐
  MOS1 MOS2  MOS3    MOS4        ← 4 个 MOS 模块的 IN/PWM 脚
  GND  GND   GND     GND  ─────  ESP32 GND  (共地, 必须)

  加热片 ── MOS 模块的输出端(V+ / OUT)
  加热电源(独立) ── MOS 模块 VIN ；电源 GND 也接 ESP32 GND
```

要点：
- **加热片用独立电源**（按你加热片的电压，比如 5V/12V），经 MOS 开关，**不要**直接从 ESP32 取电。
- **所有 GND 必须连在一起**（ESP32、加热电源、MOS 模块共地），否则 PWM 控制无效。
- IO32 一根线驱动 4 个 MOS 的栅极完全没问题（栅极几乎不耗流）。

### 3.3 接线检查清单
- [ ] DS18B20 的 DATA→IO25，且有 4.7k 上拉
- [ ] 4 个 MOS 的信号脚都接到 IO32
- [ ] ESP32 GND、加热电源 GND、MOS GND 三者共地
- [ ] 加热片由独立电源供电，不是从板子取电

## 第 4 步：烧录正式固件 + 触摸校准

1. 接好线后插 USB-C。
2. 打开 [code/03_touchscreen_ui.ino](../code/03_touchscreen_ui.ino)，点 **Upload** 烧录。
3. **首次开机会进入触摸校准**：屏幕四角会依次出现箭头，**用手指/触摸笔依次点准每个箭头**。
   - 校准结果自动存进板子（NVS），以后开机不再校准。
   - 想重新校准：在 IDE 串口里没有快捷入口，简单做法是把 `setupTouch()` 里读取逻辑临时跳过，或重刷固件前先擦除 flash（`Tools` → `Erase All Flash Before Upload` 选 Enabled 烧一次）。
4. 校准完进入主界面：温度大字 + 三个档位按钮。

## 第 5 步：调试与验收

打开串口监视器（**115200**），每秒会打印：
```
Temp:34.5C Target:34C PWM:100
```

按需求文档第 7 节逐条验收：

| 验收项 | 怎么测 | 期望 |
|--------|--------|------|
| 独立运行 | 拔掉电脑，只用充电器供电上电 | 屏幕正常显示、可触摸 |
| 触摸换档 | 依次点 30/34/38 | 目标温度变化，被点按钮高亮 |
| 加热行为 | 点 38，摸加热片/看 PWM | 温度低于目标时 PWM=255 全力加热 |
| 高温倒计时 | 点 38 等待 | 状态行显示 `Boost 14:59…` 倒数 |
| 自动降档 | 38 档等满 15 分钟（测试可临时把代码里 `BOOST_DURATION` 改小，如 60000=1分钟） | 自动回到 34，按钮高亮跳到 34 |
| 控温收敛 | 静置观察 | 温度稳定在目标附近 |
| RGB 档位灯 | 切换三档 | 板载灯变 蓝(30)/绿(34)/红(38) |

### 常见问题

| 问题 | 原因 / 处理 |
|------|------------|
| 温度显示 -127 或不变 | DS18B20 没接好或缺 4.7k 上拉；查 IO25 接线 |
| 点按钮没反应 / 点偏 | 触摸没校准好，重新校准（见第 4 步） |
| 加热片不热 | 检查共地；MOS 信号脚是否接 IO32；加热电源是否独立供上电 |
| 编译报错找不到库 | 第 1.4 步三个库没装全 |
| 上传失败 / 连不上 | 按住板子 **BOOT** 键再点 Upload，松手；或装 CH340 驱动 |

## 附：从 02 版到 03 版改了什么

| 方面 | 02 版（Arduino） | 03 版（本方案） |
|------|------------------|-----------------|
| 换档方式 | 电脑发 Serial 命令 T30/T34/T38 | 屏幕触摸按钮 |
| 显示 | 串口文字 | 3.5" 触摸屏图形界面 |
| 主控 | Arduino | ESP32-WROOM-32E（带屏板） |
| DS18B20 脚 | pin 2 | **IO25** |
| 加热脚 | pin 3/5/9/10（4 路） | **IO32（4 路并联 1 根）** |
| 控温算法 | — | **完全不变** |
| 高温自动降档 | 有 | 有，且界面实时倒计时 |
