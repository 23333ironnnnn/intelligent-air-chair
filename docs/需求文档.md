# 智能气垫椅 — 触摸屏温控 需求文档

> 版本 v1.0 ｜ 2026-06-09
> 目标:把现有"Serial 命令切换三档温控"升级为**触摸屏可视化 + 触摸调档**。

---

## 1. 背景

当前项目（[02_three_level_control.ino](../code/02_three_level_control.ino)）已实现：

- DS18B20 读温度
- 三档目标温度：低 30°C / 中 34°C / 高 38°C
- 高温档运行 15 分钟自动降到中温档
- 4 路 MOS 模块 PWM 加热
- **切换档位靠电脑发 Serial 命令（T30/T34/T38）** ← 痛点：脱离电脑就没法操作

本次升级用一块带屏的 ESP32 开发板，让用户**直接在屏幕上看温度、点按钮换档**，脱离电脑独立运行。

---

## 2. 硬件清单

| 部件 | 型号 / 规格 | 说明 |
|------|------------|------|
| 主控带屏板 | **Sunton ESP32-3248S035R**（黄色屏 / CYD） | ESP32-WROOM-32E + 3.5" ST7796 320×480 + XPT2046 电阻触摸 |
| 温度传感器 | DS18B20 | 原项目已有 |
| 加热驱动 | MOS 管模块 ×4 | 原项目已有，本方案 4 路栅极并联 |
| 加热元件 | 发热片 / 加热垫 | 气垫椅本体 |
| 电阻 | 4.7kΩ | DS18B20 上拉 |
| 电源 | 加热元件独立供电 + USB-C 给板子 | 注意共地 |

> ⚠️ 重要：板子丝印写的是 `ESP32-32E`、`3.5" 320x480`、`Resistance Touch`，**不是 ESP32-S3、不是电容触摸**。商家资料页给的是另一块板（2.8" S3），不要照那个接。

## 3. 功能需求

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1 | 实时温度显示 | 屏幕大字显示当前温度，每秒刷新 | 必须 |
| F2 | 触摸调档 | 屏幕上三个按钮：30 / 34 / 38°C，点击即切换 | 必须 |
| F3 | 当前档高亮 | 当前所选档位按钮高亮，一眼能看出在哪档 | 必须 |
| F4 | 加热状态显示 | 显示目标温度和当前 PWM 输出强度 | 必须 |
| F5 | 高温档倒计时 | 高温档显示剩余时间，15 分钟到自动降中档并刷新界面 | 必须 |
| F6 | 控温逻辑不变 | 沿用 02 版算法（低于目标-0.5 全功率 / 达标停 / 中间细调） | 必须 |
| F7 | 触摸校准 | 电阻屏首次开机引导点四角校准，结果保存，下次免校准 | 必须 |
| F8 | Serial 调试输出 | 仍保留串口打印，方便排查 | 建议 |
| F9 | 板载 RGB 档位灯 | 用板载 RGB 灯指示档位：低=蓝/中=绿/高=红，不看屏也知道在哪档 | 建议 |

---

## 4. 非功能需求

- **独立运行**：上电即工作，不依赖电脑。
- **温度刷新**：≥ 1 次/秒。
- **触摸响应**：点击到换档 < 0.5s。
- **安全**：加热元件用独立电源经 MOS 开关，ESP32 只出控制信号；必须共地。

## 6. 引脚分配（最终方案）

> ✅ 以下已与官方资料核对一致：https://www.lcdwiki.com/zh/3.5inch_ESP32-32E_Display

板子上**唯一空闲、且能做输出**的引脚在 **I2C 4pin 座子**（`3.3V / IO32(SDA) / IO25(SCL) / GND`）。
`IO35/IO39` 座子是只能输入的脚，不能用来驱动加热。

| 用途 | ESP32 引脚 | 来源 |
|------|-----------|------|
| DS18B20 数据 | **IO25** | I2C 座子（SCL） |
| 4 路 MOS 加热（栅极并联） | **IO32** | I2C 座子（SDA） |
| 传感器/MOS 供电 3.3V、GND | 3.3V / GND | I2C 座子 |
| 屏幕 ST7796U（板载固定） | SCLK14/MOSI13/MISO12/CS15/DC2/BL27，RST 接 EN | 板载 |
| 触摸 XPT2046（板载固定） | CS33/IRQ36 + 共用上面的 SPI | 板载 |
| 板载 RGB 档位灯（板载固定） | 红IO22 / 绿IO16 / 蓝IO17（共阳，LOW 点亮） | 板载 |
| SD 卡（未使用） | CS5/CLK18/MOSI23/MISO19 | 板载 |
| 音频（未使用） | 使能IO4 / DAC输出IO26 | 板载 |

> **为什么 4 路加热并到一根线？** 02 版代码里 4 路 MOS 始终输出**相同** PWM，本质是同步的；而这块板没有 4 个空闲输出脚。把 4 个 MOS 模块的信号输入端并联到 IO32，一根 PWM 同时驱动，效果完全等价、且省脚。

## 7. 验收标准

1. 不接电脑，上电后屏幕显示温度并能触摸换档。
2. 点 30/34/38 三个按钮，目标温度和加热行为随之改变，当前档高亮正确。
3. 点 38°C 后出现倒计时，15 分钟后自动回到 34°C 并刷新界面。
4. 实测温度能稳定收敛到目标附近（沿用 02 版已验证的算法）。
